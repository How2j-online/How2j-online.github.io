三、简单的概念

工作目录 暂存区 版本库
包含.git文件夹的目录。 .git文件中的index文件就是暂存区，也可以叫stage，临时保存修改文件的地方。 .git文件夹存储配置信息、版本信息、日志信息。 
graph LR
A(工作目录)
-->| git add| B(暂存区)
-->| git commit| C(版本库)

文件状态

  1. untrackde 未跟踪(为纳入版本控制)
  2. tracked 以跟踪(被纳入版本控制)
    • Unmodified 未修改状态
    • Modified 已修改状态
    • Staged 已暂存转态

五、远程仓库操作

  1. git remote 如果已经配置远程仓库服务器，会列出指定的每一个远程服务器简写，
    如果是克隆，则显示origin。

  2. git remote add <shortname> <url> 添加一个新的git仓库，同时指定一个可以引用的简写
    git clone 克隆

  1. git remote rm 移除仓库(本地)

    • 从远程仓库中抓取和拉取
    • git fetch 获取(不能merge，即不能自动合并数据，需要手动合并，使用命令git merge <shortname> <mastername>，如git fetch origin master)
    • git pull 获取(能merge，git pull <shortname> <mastername>)
    • 当本地仓库存在文件时，拉取是会报错futal:refusing to merge unrelated histories，需要在后加入参数--allow-unrelated-histories

  2. 推送到远程仓库
    git push <remote-name> <branch-name>

六、git分支

  1. 查看分支

    • git branch 列出本地所有分支
    • git branch -r 列出远程仓库的所有分支
    • git branch -a 列出本地和远程的分支

  2. 创建分支

    • git branch [分支名称]

  3. 切换分支和分支推送

    • git checkout [分支名称]
    • git push [远程服务器名称] [分支名称]

  4. 分支合并

    • git merge [分支名称]
    • 如果在两个不同的分支中，对同一个文件进行了修改，合并时，会提示冲突，需要手动删除不需要的内容，然后使用git add命令标识冲突解决。

  5. 删除分支

    • git branch -d [分支名称]
    • git branch -D [分支名称] 强制删除

七、git标签

  1. 创建标签

    • git tag [tagName] 创建新标签
    • git tag 列出所有tag
    • git show [tagName] 查看tag信息

  2. 标签推送和检出标签

    • git push [远程仓库名] [tagName]
    • git checkout -b [分支名] [tagName] 新建一个分支，指向莫个标签

  3. 删除标签

    • git tag -d [tagName] 删除了本地
    • git push [远程仓库名] :refs/tags/[tagName] 删除远程
