this的绑定规则

  1. 函数在调用时，js会默认给this绑定一个值。
  2. this的绑定和定义的文字没有任何关系。
  3. 与调用方式以及调用位置有关。
  4. this是在运行时进行绑定的，而不是在编写时绑定，它的上下文取决于函数调用时的各种条件。this的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式。

默认绑定

函数独立调用时，会使用默认绑定，this指向全局对象。

隐式绑定

函数作为对象的方法调用时，this指向该对象。

显示绑定

使用call、apply、bind方法，this指向绑定的对象。

call和apply的区别在于传入的参数不同，call接受参数列表，apply接受参数数组。
bind方法会创建一个新函数，称为绑定函数，当调用这个函数时，绑定函数会以创建它时传入bind方法的第一个参数作为this，传入bind方法的第二个以及以后的参数加上绑定函数运行时本身的参数按照顺序作为原函数的参数来调用原函数。

new绑定

使用new调用函数时，this指向新创建的对象。
当一个函数使用new来调用时，我们会构造一个新的对象并把它绑定到函数调用的this上。

  1. 创建一个新对象
  2. 这个新对象会被执行[[原型]]连接
  3. 这个新对象会绑定到函数调用的this上
  4. 如果函数没有返回其他对象，那么new表达式中的函数调用会自动返回这个新对象

内置函数绑定

内置函数的this指向全局对象或者undefined。

this绑定优先级

  1. new绑定 > 显示绑定 > 隐式绑定 > 默认绑定
  2. new绑定和显示绑定是显式绑定，this指向绑定的对象
  3. 隐式绑定和默认绑定是隐式绑定，this指向调用位置的上下文对象或者全局对象
  4. new绑定和隐式绑定是优先级最高的，new绑定会把this绑定到新创建的对象上，隐式绑定会把this绑定到调用位置的上下文对象上

显示之外的绑定

  1. 显示绑定null或者undefined，规则会使用默认绑定，this指向全局对象, 严格模式下指向null或者undefined
  2. 间接函数引用(obj1.foo = obj2.foo)()

链表

链表是一种线性表，但是并不会按线性的顺序存储数据，而是在每一个节点里存到下一个节点的指针(Pointer)。

  • 不必在创建时确定大小
